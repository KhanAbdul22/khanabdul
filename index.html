Deliverables

Designs for the following pages and sections:

  • Global navigation and Footer
  • Homepage
  • Article page layouts (2) with various styles
  • Category page (1 template/4 pages–Sense/Matter/Space/Future)
  • Contributor/Author Page

In addition to designs for various page layouts, we’ll also think through:

  • Interactive states (how the page loads, how things respond to a user’s cursor)
  • How the website behaves on various screen sizes.

We will provide the following, as necessary:

  • static comps of layouts for the above pages, states, scenarios
  • prototypes (click-through or animated)
  • asset templates
  • interactive states
  • styles, documentation, annotation for development

Art Direction, Content Creation, and Content Entry

We think of this phase in three parts:

  • Direction – Are the assets you’re using tell the story you want to tell? Will they work with the new designs?
  • Creation – Will we want to or need to make anything new for the website?
  • Content entry – A large of amount of the content will most likely be able to be migrated from the current site. (This is everything from editorial content to product info to sales data.) However, there might be some new content that needs to be entered manually. Who will do that?

At the minimum, we will need to enter some copy for interface elements and clean-up migrated content.

Technology and Development

During the development phase, we will share staging URLs and CMS access. We’ll provide structure for submitting feedback for 2 rounds of testing.

We will be delivering:

  • Fully functioning website reflecting approved designs
  • Github repo with website codebase
  • CMS to host and update editable website content
  • Best practices for modern web development standards and technologies, SEO, AA-level accessibility, Social Graph metadata, etc.
  • Documentation for how the codebase is set up (via Github) and any guidelines for using the CMS (as necessary).
  • Integrations for any third-party services (e.g. analytics) or software (e.g. webfonts) as per scope
